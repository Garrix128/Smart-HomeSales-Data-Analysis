# 智能家居销量数据分析系统 - 项目说明

## 项目概述

本项目是一个基于Hadoop的智能家居销量数据分析系统，采用前后端分离架构，实现了从数据清洗、分析到可视化的完整流程。

## 技术架构

### 前端技术栈
- **Vue 3.3.4**: 渐进式JavaScript框架
- **Element Plus 2.3.12**: Vue 3组件库
- **ECharts 5.4.3**: 数据可视化图表库
- **Vue Router 4.2.4**: 路由管理
- **Axios 1.5.0**: HTTP客户端

### 后端技术栈
- **SpringBoot 2.7.18**: Java Web框架
- **MyBatis Plus 3.5.3.1**: ORM框架
- **MySQL 5.6.x**: 关系型数据库
- **JDK 1.8**: Java开发工具包

### 数据处理技术栈
- **PySpark 3.4.0**: 大数据处理框架
- **Hadoop 3.3.6**: 分布式存储系统
- **Python 3.10**: 编程语言

## 核心功能模块

### 1. 数据清洗模块 (data_cleaning.py)
- 读取5个Excel数据源文件
- 数据清洗：处理缺失值、异常值、重复数据
- 数据标准化：日期格式、数值类型、分类变量
- 数据关联：通过订单编号、产品型号等字段关联
- 数据存储：保存清洗后的数据到HDFS

### 2. 数据分析模块 (data_analysis.py)
- **销售趋势分析**: 按时间序列分析销售额变化
- **地域分布分析**: 分析各省份销售分布情况
- **用户画像分析**: 会员与非会员购买行为对比
- **产品品类分析**: 各品类销售额和销量排名
- **随机森林模型**: 构建预测模型，分析特征重要性

### 3. 后端API模块
提供6个Controller，共20+个API接口：
- **SalesDataController**: 销售数据查询
- **AnalysisResultController**: 分析结果获取
- **UserBehaviorController**: 用户行为分析
- **ProductAnalysisController**: 产品分析
- **PredictionController**: 预测结果
- **DataManageController**: 数据管理（CRUD）

### 4. 前端可视化模块
包含6个可视化图表组件：
- **SalesTrendChart**: 销售额时间趋势图（折线图）
- **RegionMap**: 地域分布热力图（地图）
- **ProductAnalysis**: 产品品类占比图（饼图）
- **ChannelAnalysis**: 销售渠道分析图（柱状图）
- **UserProfileChart**: 用户画像分析图（雷达图）
- **RandomForestChart**: 随机森林特征重要性图（条形图）

## 数据库设计

系统包含6个核心数据表：

1. **sales_summary**: 销售汇总表
2. **region_analysis**: 地域分析表
3. **user_behavior**: 用户行为分析表
4. **product_analysis**: 产品分析表
5. **prediction_result**: 模型预测结果表
6. **feature_importance**: 特征重要性表

## 数据流程

```
原始Excel数据
    ↓
PySpark数据清洗 (data_cleaning.py)
    ↓
HDFS存储 (hdfs://localhost:9000/smart_home/cleaned_data/)
    ↓
PySpark数据分析 (data_analysis.py)
    ↓
MySQL结果存储
    ↓
SpringBoot API
    ↓
Vue前端展示
```

## 快速开始

### 1. 环境准备
- 安装JDK 1.8
- 安装Maven 3.9.9
- 安装Python 3.10
- 安装Node.js 16+
- 安装MySQL 5.6.x
- 安装Hadoop 3.3.6
- 安装Spark 3.4.0

### 2. 数据库初始化
```bash
mysql -u root -p < Smart_Home_Analysis/src/main/resources/db/schema.sql
```

### 3. 启动后端
```bash
cd Smart_Home_Analysis
mvn clean package
java -jar target/Smart_Home_Analysis-1.0-SNAPSHOT.jar
```

### 4. 启动前端
```bash
cd Smart_Home_Analysis/frontend
npm install
npm run serve
```

### 5. 运行数据分析
```bash
cd python_analysis
pip install -r requirements.txt
python main.py
```

### 6. Docker部署（可选）
```bash
docker-compose up -d
```

## 访问地址

## 注意事项

1. **Hadoop环境**: 如果HDFS不可用，数据会自动保存到本地
2. **MySQL配置**: 需要修改 `application.yml` 中的数据库连接信息
3. **跨域配置**: 已通过CorsConfig配置，支持跨域请求
4. **数据文件**: 确保5个Excel文件已放入 `python_analysis/data/` 目录
5. **端口占用**: 确保8080和8081端口未被占用
