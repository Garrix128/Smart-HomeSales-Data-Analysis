# 数据库配置说明

## 错误原因
`Access denied for user 'root'@'localhost'` 表示MySQL密码不正确。

## 解决方案

### 方法1: 修改配置文件中的密码（推荐）

编辑 `src/main/resources/application.yml`，将密码改为你的MySQL实际密码：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/smart_home?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: 你的MySQL密码  # 修改这里
```

### 方法2: 检查MySQL密码

#### Windows系统：
1. 打开命令提示符（CMD）或PowerShell
2. 测试MySQL连接：
```bash
mysql -u root -p
```
输入密码，如果能连接成功，说明密码正确。

#### 如果忘记MySQL密码：
1. 停止MySQL服务
2. 以跳过权限表的方式启动MySQL
3. 重置密码

### 方法3: 创建新的MySQL用户（可选）

如果不想使用root用户，可以创建新用户：

```sql
-- 登录MySQL
mysql -u root -p

-- 创建新用户
CREATE USER 'smarthome'@'localhost' IDENTIFIED BY 'smarthome123';

-- 授予权限
GRANT ALL PRIVILEGES ON smart_home.* TO 'smarthome'@'localhost';

-- 刷新权限
FLUSH PRIVILEGES;
```

然后修改 `application.yml`:
```yaml
spring:
  datasource:
    username: smarthome
    password: smarthome123
```

## 常见MySQL密码

如果不知道密码，可以尝试：
- `root`
- `123456`
- `password`
- `admin`
- 空密码（留空）

## 验证配置

修改密码后，重新启动后端服务，应该能够成功连接数据库。
