# 系统启动说明

## 问题排查

如果遇到 `ECONNREFUSED` 错误，说明后端服务没有启动。

## 启动步骤

### 1. 确保MySQL数据库已启动
- MySQL服务必须运行
- 数据库 `smart_home` 必须存在
- 用户表 `sys_user` 必须存在（运行 `python_analysis/00_初始化用户表.py` 创建）

### 2. 检查数据库配置

编辑 `src/main/resources/application.yml`，确保数据库配置正确：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/smart_home?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: root  # 修改为你的MySQL密码
```

### 3. 启动后端服务

#### 方式1: 使用IDE启动（推荐）
1. 打开 `SmartHomeApplication.java`
2. 右键 -> Run 'SmartHomeApplication.main()'

#### 方式2: 使用Maven命令启动
```bash
cd Smart_Home_Analysis
mvn spring-boot:run
```

#### 方式3: 打包后启动
```bash
cd Smart_Home_Analysis
mvn clean package
java -jar target/Smart_Home_Analysis-1.0-SNAPSHOT.jar
```

### 4. 验证后端启动成功

启动成功后，你应该看到：
```
Started SmartHomeApplication in X.XXX seconds
```

访问测试：
- http://localhost:8080/api/auth/login (应该返回错误，但说明服务已启动)

### 5. 启动前端服务

```bash
cd Smart_Home_Analysis/frontend
npm run serve
```

## 常见问题

### 问题1: 端口被占用
如果8080端口被占用，修改 `application.yml`:
```yaml
server:
  port: 8081  # 改为其他端口
```

同时修改 `frontend/vue.config.js`:
```javascript
proxy: {
  '/api': {
    target: 'http://localhost:8081',  // 改为对应的端口
    ...
  }
}
```

### 问题2: 数据库连接失败
- 检查MySQL服务是否启动
- 检查数据库用户名和密码是否正确
- 检查数据库 `smart_home` 是否存在

### 问题3: 用户表不存在
运行Python脚本创建用户表：
```bash
cd python_analysis
python 00_初始化用户表.py
```

## 启动顺序

1. **启动MySQL数据库**
2. **初始化用户表**（首次运行）
   ```bash
   python python_analysis/00_初始化用户表.py
   ```
3. **启动后端服务**
   ```bash
   # 在IDE中运行 SmartHomeApplication
   # 或使用 mvn spring-boot:run
   ```
4. **启动前端服务**
   ```bash
   cd Smart_Home_Analysis/frontend
   npm run serve
   ```

## 验证

1. 后端：访问 http://localhost:8080 （应该看到错误页面，说明服务运行）
2. 前端：访问 http://localhost:8081 （应该看到登录页面）
3. 登录：使用 admin/admin 登录
